# 开发规范

## 代码风格

### C++

- 遵循 Qt 编码规范
- 类名：`PascalCase`
- 方法名：`camelCase`
- 成员变量：`m_camelCase`
- 常量：`UPPER_SNAKE_CASE`

### QML

- 组件文件名：`PascalCase.qml`
- 属性名：`camelCase`
- 信号名：`camelCase`，动词开头

## 文件组织

```
src/
├── core/        # 业务逻辑（无 UI 依赖）
├── models/      # 数据模型
├── services/    # 外部服务集成
├── ui/          # QML 文件
└── utils/       # 工具类
```

## 提交规范

使用 Conventional Commits：

```
feat: 新功能
fix: 修复 bug
docs: 文档更新
refactor: 重构
test: 测试相关
chore: 构建/工具链
```

## 分支策略

- `main`：稳定版本
- `dev`：开发主线
- `feature/xxx`：功能分支
- `fix/xxx`：修复分支

## 代码审查清单

- [ ] 无内存泄漏
- [ ] 异步操作有错误处理
- [ ] 敏感信息（API Key）不硬编码
- [ ] 新功能有对应测试
