# 版本管理

## 版本号规范

采用语义化版本 `MAJOR.MINOR.PATCH`：

- **MAJOR**：不兼容的 API 变更
- **MINOR**：向下兼容的新功能
- **PATCH**：向下兼容的 bug 修复

## 发布流程

1. 从 `dev` 创建 `release/vX.X.X` 分支
2. 更新版本号与 CHANGELOG
3. 测试通过后合并至 `main`
4. 打 Tag，构建发布包

## CHANGELOG 格式

```markdown
## [1.0.0] - YYYY-MM-DD

### Added
- 新增功能描述

### Fixed
- 修复问题描述

### Changed
- 变更描述
```
